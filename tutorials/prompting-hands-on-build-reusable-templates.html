Prompting: Hands-on Templates

Treat prompts like small products: inputs, outputs, examples, and failure handling. You’ll convert a one-off prompt into a reusable template with a test set.

What you will build

  • One reusable template with variables and a fixed schema.
  • At least one positive and one messy example.
  • A short “what to do when it fails” playbook.

Prereqs

  • Prompting: Structure Quickstart

Concept explainer: template ingredients

  • Variables: placeholders you swap (e.g., {{text}}, {{audience}}).
  • Output schema: required keys/sections; keeps outputs comparable.
  • Examples: one good example + one messy example beats long rules.

Worked example: JSON extractor

Template: “Extract entities from {{text}}. Output JSON with keys: people, orgs, dates. If none, return an empty array for that key. Keep explanations under 12 words.”

Add a messy example (typos, mixed casing) to show intended behavior.

Guided mini-build: your template

  1. Pick a task (summary, extraction, rewrite, classification).
  2. Define variables and the schema (JSON or table headers).
  3. Write one clean example and one messy example.
  4. Write a failure playbook: if output is missing a key → re-run with “ensure keys exist”.

Practice Lab: Run the template

Task: Run the template on 5 inputs (include the messy one) and refine once.

  1. Execute on 5 inputs; note any schema breaks.
  2. Apply the failure playbook once; re-run the failing input.
  3. Record before/after outputs to confirm the tweak.

Success criteria:

  • Outputs are structurally consistent across 5 inputs.
  • Messy input still yields usable output.
  • Failure playbook documented and applied at least once.

Recap

  • Variables + schema + examples = repeatable template.
  • Messy examples surface fragility early.
  • Write the recovery step once; reuse it instead of ad-hoc retries.
