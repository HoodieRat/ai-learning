Hardware Capstone: Quantize and Fit a Model

Your deliverable is a single-page “fit and tradeoff” note: the model you want, the quantization you chose, the memory it needs, and what quality you lose (if any).

Prereqs

  • Hardware: Pitfalls (Bottlenecks & Thermals)
  • Hardware: Quantization Explained

Artifact

  • Chosen model + chosen quantization format (what and why)
  • Observed performance before/after (simple timing is fine)
  • Observed quality change on a small test set

Plan your run

  1. Pick a model that barely fits (or almost fits) your GPU/CPU budget.
  2. Choose two quantization options: one conservative, one aggressive.
  3. Define a 10-prompt quality tripwire (reuse from Quantization Explained if you made one).
  4. Decide on metrics: tokens/sec (or images/min) and a simple pass count on the tripwire.

Mini build: run and compare

  1. Run the baseline precision once; record throughput and whether the model fits without paging.
  2. Run the conservative quantized build; record the same metrics and any quality changes.
  3. Run the aggressive build; note if fit improves and where quality drops.
  4. Pick the best option; write a one-paragraph "why this wins" tied to fit, speed, and quality.

Practice Lab

Task: Make one model fit better on your hardware.

Do: Select a model that is borderline for your memory budget and apply quantization or a smaller variant.

Success criteria:

  • The model runs without memory paging crashes.
  • You can demonstrate a measurable improvement (speed or stability).
  • You record at least one quality regression test (a short prompt set).

Verify: Run your test set before and after; document the one change that mattered most.

Recap

  • Quantization is the fastest lever to regain fit; track speed and quality together.
  • Decide using a tiny, repeatable test set that mirrors your workload.
  • Document what changed so you can redo the experiment on future hardware.
