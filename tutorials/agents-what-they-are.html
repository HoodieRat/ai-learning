A grounded definition of agents, tools, loops, and the situations where simple prompts are better.

Core concepts

  • Planner, router, tools: separate thinking, choosing, doing.
  • Observation-action loop needs guardrails and timeouts.
  • Humans stay in the loop to approve high-risk steps.

Worked example

  1. Define goal -> plan -> choose tool -> act -> reflect.
  2. Set a max step count and time budget.
  3. Log each observation/action pair for replay.

Mini build

  1. Write a tiny loop with planner->tool->reflect.
  2. Add a max-steps and timeout guard.
  3. Log each step to a file for replay.

Practice Lab

Task: Run a 3-step loop on a toy task; capture steps and ensure it stops safely.

Success criteria:

  • You capture parameters and outputs for the run.
  • You can repeat the run and get consistent behavior.
  • You note at least one risk or failure mode to watch.

Recap

  • Loops need guardrails and logs.
  • Separate planning from acting.
  • Humans approve high-risk steps.
