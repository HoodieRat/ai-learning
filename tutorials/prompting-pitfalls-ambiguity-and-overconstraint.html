Prompting: Pitfalls (Ambiguity & Overconstraint)

Prompts fail by being too vague (model guesses) or overconstrained (rules conflict). Learn the smells, the fixes, and how to prove the fix works.

What you will learn

  • How to identify ambiguity vs conflicting constraints.
  • How to prioritize “must” vs “nice-to-have”.
  • How to use positive and negative examples to lock behavior.

Prereqs

  • Prompting: Hands-on Templates

Concept explainer: two failure classes

  • Ambiguity: missing audience, objective, success criteria.
  • Overconstraint: conflicting rules (“be brief” and “include all details”).
  • Uncheckable asks: no schema/examples to judge against.

Worked example: fixing both

Before (vague): “Summarize this for me.”

After (fixed): “Summarize for executives. Output JSON {decisions:[], risks:[], open_questions:[]}. Max 3 items each. If info missing, set to []. Include one source quote.”

Overconstraint fix: drop conflicting tone/length rules; keep one primary constraint (schema) and one secondary (max words).

Guided mini-build: diagnose and rewrite

  1. Pick a failing prompt; label it “ambiguous” or “overconstrained”.
  2. Add/clarify the deliverable (schema/table/steps) and audience.
  3. List constraints; keep the top 3 “musts”; remove or soften the rest.
  4. Add one positive example and one negative example (what not to do).

Practice Lab: Debug a prompt

Task: Rewrite and prove the fix.

  1. Run the original prompt on one input; save the output.
  2. Apply your rewrite (schema + prioritized constraints + examples).
  3. Run the same input; compare: format consistency, missing info, conflicts resolved.
  4. If one issue remains, adjust one constraint or example and re-run once.

Success criteria:

  • Rewritten prompt has a fixed output format.
  • At least one conflicting constraint removed or clarified.
  • New output is easier to verify and closer to the intended shape.

Recap

  • Decide the deliverable first; constraints serve it.
  • Prioritize must-haves; avoid dueling rules.
  • Use examples to lock behavior; one negative example is powerful.
