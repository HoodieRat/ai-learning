Agentic Flows Capstone: Guardrailed Agent Prototype

You’ll design a small agent prototype that is operable and safe: budgets, logging, verification gates, and an incident playbook.

Prereqs

  • Agentic Flows: Pitfalls (Tool Failures & Loops)

Artifact

  • Tool catalog + schemas
  • Budgets + stop conditions
  • Logging/redaction plan
  • Verifier gate before external actions

Plan your prototype

  1. Pick a low-risk automation (status drafting, FAQ answer, changelog summary).
  2. List tools with schemas and which are read vs write.
  3. Set numeric budgets (tool calls, retries, time) and stop conditions.
  4. Define verification gates for every state-changing action.

Mini build: run a tabletop

  1. Simulate one happy-path run; capture logs you would keep.
  2. Simulate two failure cases (tool timeout, bad data) and note the halt/escalation.
  3. Ensure redaction rules are applied in the logs.
  4. Write the incident steps: who gets alerted, what gets rolled back.

Practice Lab

Task: Produce a “runbook-ready” agent design.

Do: Pick one automation (safe and low-risk). Specify tools, budgets, and verification gates. Write 3 failure scenarios and your responses.

Success criteria:

  • The design can be operated by someone else with your notes.
  • There is a clear escalation path for failures or uncertainty.
  • All state-changing actions require verification (and optionally human review).

Verify: Walk through a failure scenario and confirm the agent ends in a safe state.

Recap

  • Guardrails = constrained tools, budgets, verifiers, and clear escalation.
  • Tabletop simulations surface missing logs and redactions.
  • Document the incident path so operators know how to halt safely.
