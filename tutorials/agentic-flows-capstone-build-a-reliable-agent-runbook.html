Artifact

  • Tool catalog with schemas
  • Logging/redaction plan
  • Budgets and stop conditions
  • Failure playbook (what to do when stuck)

Plan your runbook

  1. Choose one agent use case and its goal.
  2. List allowed tools (read vs write) with parameter schemas.
  3. Set numeric budgets and stop conditions for loops and retries.
  4. Define escalation: when to involve a human and how to pause the agent.

Mini build: incident walkthrough

  1. Write three common failure modes (tool timeout, bad data, verifier fail).
  2. For each, specify logs to capture, what to redact, and who is notified.
  3. Include rollback/containment steps for any state changes.
  4. Test one scenario as a tabletop and adjust the runbook where unclear.

Practice Lab

Task: Write an operational runbook.

Do: Choose one agent use case and produce a one-page runbook covering: inputs, outputs, tools, budgets, and review gates.

Success criteria:

  • The runbook is specific enough someone else could operate it.
  • There is a clear escalation path (human review or rollback).
  • It includes at least 3 known failure modes and mitigations.

Verify: Walk through a hypothetical incident and confirm the runbook provides steps.

Recap

  • Runbooks make agents operable: clear tools, budgets, and logs.
  • Escalation and rollback steps keep incidents contained.
  • Tabletop drills reveal missing details before production.
