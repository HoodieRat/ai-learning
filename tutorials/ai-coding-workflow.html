How to use AI on real projects: diffs, constraints, and verifying changes without chaos.

Core concepts

  • Always ask for diffs, not whole files; review before apply.
  • Tests and linters catch regressions; run them after each patch.
  • Log assumptions and risky areas to validate manually.

Worked example

  1. Provide file+line context and ask for a minimal diff.
  2. Run tests; if failing, feed logs back for a second patch.
  3. Summarize risk areas before merging.

Mini build

  1. Create a prompt template for diffs with tests.
  2. Wire a script to apply patches and run tests.
  3. Store run logs and summaries per attempt.

Practice Lab

Task: Apply an AI-generated diff to a small repo, run tests, and note failures.

Success criteria:

  • You capture parameters and outputs for the run.
  • You can repeat the run and get consistent behavior.
  • You note at least one risk or failure mode to watch.

Recap

  • Ask for diffs; run tests.
  • Log assumptions and risks.
  • Review before merging.
