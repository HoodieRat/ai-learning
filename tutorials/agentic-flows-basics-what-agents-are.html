Agentic Flows: What Agents Are

Agents plan and act with tools. The upside is automation; the risk is uncontrolled behavior. This tutorial defines the core parts, shows a minimal loop, and adds the safety rails.

Prereqs

  • AI 101: What Models Do
  • Evaluation: Metrics vs Judges

Core parts

  • Goal: what success looks like.
  • Planner: breaks the goal into steps.
  • Tools: actions in the real world (APIs, files, browsers).
  • Memory: what it remembers (and what it shouldn’t).
  • Verifier: checks outputs before acting.

Worked example: a safe weekly status bot

  1. Goal: draft a weekly status from recent tickets.
  2. Planner: read tickets, extract updates, summarize.
  3. Tools: read_tickets, summarize_text. No write tools.
  4. Memory: keep only ticket IDs and summaries for this run.
  5. Verifier: check for PII and missing ticket owners before output.

Mini build: design a loop on paper

  1. Pick a low-risk automation (e.g., summarize changelog, draft status).
  2. List allowed tools and what each can change (read-only vs write).
  3. Write 2–3 stop conditions (budget reached, no progress, verifier fail).
  4. Add one verification gate before any external action.

Practice Lab

Task: Design an agent loop on paper.

Do: Choose a safe automation (e.g., summarize new docs, draft a weekly status). Define: goal, tools, stop conditions, and verification steps.

Success criteria:

  • You list the tools the agent may use.
  • You define at least 2 stop conditions.
  • You include a verifier step before any external action.

Verify: Simulate one run manually and check whether the stop conditions trigger.

Check your understanding

Recap

  • Agents = goal + planner + tools + memory + verifier + stop rules.
  • Design for safety first: restrict tools, add budgets, and verify before acting.
  • Paper simulations reveal missing stop conditions before any code runs.
