Coding AI Capstone: Ship a Small Feature with Tests

You’ll plan, implement, and verify a small end-to-end feature using AI — while keeping code reviewability and safety as first-class requirements.

Prereqs

  • Coding AI: Pitfalls

Artifact

  • A small feature (one user story)
  • Tests that prove the behavior
  • A short “verification notes” writeup

Plan your feature

  1. Write the user story and acceptance criteria (including one edge case).
  2. Break into 3–4 steps; ask AI for a plan but keep changes small.
  3. Decide tests per acceptance criterion before coding.
  4. Note any security or data-handling requirements.

Mini build: deliver and verify

  1. Implement step 1; run tests; review diff for unintended changes.
  2. Add the next step; expand tests with at least one edge/abuse case.
  3. Write a short verification note: what you checked, what still worries you.
  4. Ask a teammate (or AI) to critique the tests; adjust if gaps appear.

Practice Lab

Task: Deliver a feature with verification notes.

Do: Write a user story and acceptance criteria. Ask AI for a step plan, implement in small steps, and verify with tests.

Success criteria:

  • Acceptance criteria are met and tested.
  • At least one edge case is covered.
  • You can explain what you verified and how.

Verify: Have someone else read your tests and confirm they match the user story.

Recap

  • Acceptance criteria drive both code and tests; keep them explicit.
  • Ship in small, verifiable steps; review diffs every step.
  • Verification notes make review faster and safer.
